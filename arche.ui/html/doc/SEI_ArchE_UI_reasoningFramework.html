ArchE Reasoning Framework

Identifier:
SEI.ArchE.UI.reasoningFramework

Description:
This extension point is used to define a reasoning framework plug-in for ArchE. A reasoning framework definition consists of:
  • scenario types, including the definitions for the scenario parts
  • parameter types
  • parameter types that are required by responsibilities
  • relationship types
  • parameter types that are required by each relationship type
  • model element types
  • model relation types

A reasoning framework plug-in will typically contain the following files (example):

  • plugin.xml: using the reasoningFramework extension point
  • build.properties: standard Eclipse PDE build.properties including all files
  • rfconfig.xml: central piece of the reasoning framework plug-in is the configuration xml file. This file can be produced using ArchE Configurator
  • LoadAllRules.clp: file indicated in the clpFile attribute of the extension point. It is loaded in Jess when ArchE UI finds this reasoning framework plug-in.
  • AdditionalRules.clp: there can be zero or more other clp files that are loaded by the clp file indicated in the clpFile attribute.
  • other files used by the rules can also be packaged


Configuration Markup:

<!ELEMENT extension (rfconfigFile , clpFile)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>

  • point - A fully qualified identifier of the target extension point
  • id - An optional identifier of the extension instance
  • name - An optional name of the extension instance

<!ELEMENT description (#CDATA)>

An optional subelement whose body should contain text providing short description of the reasoning framework.



<!ELEMENT clpFile EMPTY>

<!ATTLIST clpFile

fileName CDATA #IMPLIED>

  • fileName - Pathname to the ".clp" file to be loaded for this reasoning framework. Path must be relative within this plug-in. If this reasoning framework uses several clp files, then the one indicated in this attribute should load all the others.

<!ELEMENT rfconfigFile EMPTY>

<!ATTLIST rfconfigFile

fileName CDATA "rfconfig.xml">

  • fileName - Pathname to the xml file that has the RF configuration. Path must be relative within this plug-in.

Examples:
The following is an example of the extension point: 

   
<extension 
       point=
"SEI.ArchE.UI.reasoningFramework"
>
 
      
<rfconfigFile fileName=
"rfconfig.xml"
/>

      
<clpFile fileName=
"LoadModifRF.clp"
/>

   
</extension>

As indicated, the example reasoning framework plug-in must contain two files at the root level:
  • rfconfig.xml: created using ArchE Configurator
  • LoadModifRF.clp: Jess clp file that loads the rules that are specific to this reasoning framework.
    • This clp file can assume the generic core rules will be loaded first.
    • This clp file can assume a variable *dir* will be defined in Jess pointing to folder where this file is located.
    • This clp file may load other clp files, if necessary.


API Information:
This plug-in is not for Java classes in this version.
